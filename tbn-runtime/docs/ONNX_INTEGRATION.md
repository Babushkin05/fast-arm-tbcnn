# ONNX Integration Guide

## Overview

The TBN Runtime supports loading and executing ONNX models with ternary/binary quantization. The integration provides both standard ONNX compatibility and optimizations for quantized neural networks.

## Current Implementation Status

### Completed
- Basic ONNX model loading (file and buffer)
- Graph structure parsing
- Tensor conversion (float32, int32, int64, int8, uint8)
- Model metadata extraction
- Dummy model generation for testing

### In Progress
- Real ONNX protobuf parsing
- Quantized operator support (QLinearConv, QLinearMatMul)
- Weight quantization during loading
- Graph optimization passes

### Planned
- ONNX Runtime integration
- Custom execution provider
- Dynamic shape support
- External data file support

## API Usage

### Loading ONNX Models

```cpp
#include <tbn/tbn.hpp>

// Load from file
auto model = tbn::load_model("model.onnx");

// Load from buffer
std::vector<uint8_t> model_data = load_model_data();
auto model = tbn::load_model_from_buffer(model_data.data(), model_data.size());
```

### Running Inference

```cpp
// Create session
auto session = model.create_session();

// Set input
session.set_input("input", input_tensor);

// Run inference
session.run();

// Get output
auto output = session.get_output("output");
```

## Supported Operators

### Basic Operators
- Conv: Standard convolution
- Gemm: General matrix multiplication
- Relu: Rectified linear unit
- Add: Element-wise addition
- Mul: Element-wise multiplication

### Quantized Operators
- QLinearConv: Quantized convolution
- QLinearMatMul: Quantized matrix multiplication
- QuantizeLinear: Float to quantized conversion
- DequantizeLinear: Quantized to float conversion

## Quantization Support

### Loading Quantized Models

The runtime automatically detects quantized operators and handles them appropriately:

```cpp
// Model with QLinearConv
auto model = tbn::load_model("quantized_model.onnx");

// Weights are loaded as int8 and converted to ternary/binary
// during optimization passes
```

### Runtime Quantization

Convert float models to ternary/binary during loading:

```cpp
// Load float model
auto model = tbn::load_model("float_model.onnx");

// Apply quantization
auto quantized_model = tbn::quantize_model(model,
    tbn::QuantizationType::TERNARY);
```

## Model Requirements

### Input/Output Specifications

Models must define:
- Input shapes (static or dynamic)
- Output shapes
- Data types (float32, int8, uint8)

### Weight Formats

Supported weight formats:
- Float32 (converted during loading)
- Int8 (quantized weights)
- UInt8 (quantized weights)

## Performance Optimizations

### Graph Optimizations

Applied automatically during loading:
- Constant Folding: Pre-compute constant operations
- Operator Fusion: Merge compatible operators
- Dead Code Elimination: Remove unused nodes
- Layout Optimization: Optimize tensor layouts for SIMD

### Memory Optimizations

- Weight Packing: 16x reduction for ternary, 32x for binary
- Memory Pool: Efficient tensor allocation
- Cache Alignment: SIMD-friendly memory layout

## Testing

### Test Models

Generate test models:
```bash
cd tools
python3 create_test_onnx.py
```

This creates:
- simple_model.onnx: Basic Conv→Relu→Gemm
- quantized_model.onnx: With quantized operators
- tiny_model.onnx: Minimal test model

### Running Tests

```bash
./build/bin/tbn_tests [onnx]
```

## Integration with ONNX Runtime

### Custom Execution Provider

TBN Runtime can be used as a custom execution provider for ONNX Runtime:

```cpp
// Register TBN execution provider
Ort::Env env;
Ort::SessionOptions session_options;
session_options.AppendExecutionProvider_TBN();
```

### Model Conversion

Convert models for optimal TBN performance:
```bash
python3 tools/convert_to_tbn.py model.onnx model_tbn.onnx
```

## Troubleshooting

### Common Issues

1. **Model Loading Fails**
   - Check ONNX version compatibility
   - Verify model integrity
   - Ensure all operators are supported

2. **Inference Errors**
   - Validate input shapes
   - Check data type compatibility
   - Verify quantization parameters

3. **Performance Issues**
   - Enable graph optimizations
   - Use appropriate quantization
   - Check memory alignment

### Debug Mode

Enable detailed logging:
```cpp
tbn::Logger::set_global_level(tbn::LogLevel::DEBUG);
```
